# Anthem_project

![Hackatime Badge](https://hackatime.hackclub.com/api/v1/badge/U099LHP1QKF/PGarg111/Anthem_project)

This game is based upon the book Anthem by Ayn Rand. I created this game for a school project. The game is a 3 player game with an on-screen die. 

How the game works:
1. Player 1 will press the space bar to roll the die. The player will automatically move forward that many spaces.
2. If the player lands on an event space, a pop up will show with a description and a button informing the player on the action they will take.
3. Process repeats for player 2 and 3
4. Keep playing
5. When a player lands on or passes by a checkpoint, they will automatically stop on the checkpoint. The checkpoints have information on them as to why they are checkpoints. There is also an action for most of the checkpoints.
6. When the players reaches/passes the barrier (same logic as the checkpoint), they have to roll either a 4 or 6 to be able to break past it.
7. For each player to reach the end, they must make a perfect roll 
8. There are no winners in the game. The point is for each player to make it to individuality (the main theme of the book). After all players reach the end, a screen will pop up showing how long it took each player.

What took me the longest was laying out the board of the game. Since the images for the board weren't in the layout on a singular file, I had to screenshot each square, remove the background, import it in Godot, add the Marker2D to the square, add text to the square, and then add it to the array. This process was repeated for 29 squares.

This game gave me a lot of new information. I remember using the tween property briefly in the game I made for Daydream, but I didn't really know the exact use for it. This game forced me to understand the property to be able to create the game I envisioned. I used Claude a few times to have it explain to me how the tween property worked and give some examples of how it is used to be able to undestand it.
